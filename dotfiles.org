* Bootstrap

Script to bootstrap.

#+begin_src bash :tangle init.sh
if [ ! -f "${HOME}/.emacs.d/init.el" ]; then
    ln -s "${PWD}/emacs/init.el" "${HOME}/.emacs.d/init.el"
fi

if [ ! -f "${HOME}/.elisp-locals.el" ]; then
    echo '(setq my/secrets-directory ""' >> "${HOME}/.elisp-locals.el"
    echo '      my/ledger-directory "")' >> "${HOME}/.elisp-locals.el"
fi

** Initialization

MELPA is a package archive for Emacs.  Useful libraries are stored there.

#+begin_src emacs-lisp
(require 'package)
(add-to-list 'package-archives '("melpa" . "https://melpa.org/packages/"))
(add-to-list 'package-archives '("org" . "https://orgmode.org/elpa/") t)
(package-initialize)
(package-refresh-contents)
#+end_src

The ~use-package~ library adds a macro that provides a concise way to keep
configuration for external packages organised.  External packages will always be
downloaded if they are not found locally.

(package-install 'use-package))

(eval-when-compile (require 'use-package))
(require 'use-package-ensure)

On OS X at least, ~exec-path~ and the environment =PATH= variables are not
configured correctly.  Here they are set explicitly.

#+begin_src emacs-lisp
(defun get-shell-path ()
  (seq-uniq
   (append
    exec-path
    (split-string
     (replace-regexp-in-string
      "[\t\s\n]+$"
      ""
      (shell-command-to-string "$SHELL --login -c 'echo ${PATH}'"))
     ":"))))

(require 'subr-x)
(setenv "PATH" (string-join (get-shell-path) ":"))
(setq exec-path (get-shell-path))
#+end_src

*** Org mode

Org mode is the literate programming environment this document is written in.
It's already included in Emacs but it can be enhanced with some further
configuration.

Some values must be set before Org mode is even loaded.

#+begin_src emacs-lisp
(use-package org)
(when (version<= "9.2" (org-version))
  (require 'org-tempo))

  (setq org-enforce-todo-checkbox-dependencies t
	org-enforce-todo-dependencies t
	org-todo-keywords '((sequence "TODO(t)" "NEXT(n)" "|" "DONE(d)")
			    (sequence "WAIT(w@/!)" "|" "CANC(c@/!)")))
#+end_src

A function to go to the next heading whilst narrowing is in effect is defined to
make an Org buffer feel more akin to an Info one.

#+begin_src emacs-lisp
(defun org-next-heading-node ()
  "Move to the next heading node while narrowed."
  (interactive)
  (widen)
  (forward-char 1)
  (if (search-forward-regexp "^\*+" (point-max) t)
      (progn
	(forward-line 0)
	(org-narrow-to-subtree)
	(org-content)
	(org-show-entry))))
#+end_src

A similar one is defined to go to the previous heading.

#+begin_src emacs-lisp
(defun org-prev-heading-node ()
  "Move to the previous heading node while narrowed."
  (interactive)
  (org-narrow-to-subtree)
  (goto-char (point-min))
  (widen)
  (if (search-backward-regexp "^\*+" (point-min) t)
      (progn
	(forward-line 0)
	(org-narrow-to-subtree)
	(org-content)
	(org-show-entry))
    (progn
      (org-overview)
      (goto-char (point-min)))))
#+end_src

Keybindings should be defined to easily execute both of these functions.

#+begin_src emacs-lisp
(add-hook 'org-mode-hook
	  (lambda ()
	    (local-set-key (kbd "C-c n") 'org-next-heading-node)
	    (local-set-key (kbd "C-c p") 'org-prev-heading-node)))
#+end_src

(let ((pdf-cmd (concat "%latex " "-shell-escape "

*** Rust

Rust font locking is provided by ~rust-mode~.  The major mode also interfaces
with ~rustfmt~ so that the tool is ran on save.

#+begin_src emacs-lisp
(use-package rust-mode
  :init (setq rust-format-on-save t)
  :hook
  (rust-mode . subword-mode)
  (rust-mode . lsp))
#+end_src

Interacting with Cargo is better provided with the ~cargo-minor-mode~ package.

#+begin_src emacs-lisp
(use-package cargo
  :hook (rust-mode . cargo-minor-mode))
#+end_src
